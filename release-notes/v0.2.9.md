# GrommeR v0.2.9

Oproti **v0.2.8**:

## Opravy

- **Velké tiskové PDF (Invalid array length)** – dávky bannerů (např. 236×120 cm) už nepadají při generování. Soubory nad ~40 MB se v desktopové aplikaci zpracovávají nativně (lopdf) bez načítání celého PDF do WebView.
- **Dávkové načítání** – soubory se analyzují postupně (ne paralelně), aby se nesčítala spotřeba paměti u několika velkých PDF najednou.
- **Lehčí analýza rozměrů** – metadata stránky se čtou přes pdf.js místo plného parsování pdf-lib při importu.

## Poznámky

- Nativní cesta funguje v Tauri při měřítku **1:1** a když je známá cesta ke zdrojovému souboru (dialog / drag&drop z disku). U měřítka 1:N nebo cílového formátu se stále používá pdf-lib.
- U velkých PDF je potřeba mít zvolenou **výstupní složku** (nativní zápis neotvírá dialog pro každý soubor).
